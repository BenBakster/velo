# velo — аварийное восстановление 2026-06-14

## Что случилось
При очистке Wi-Fi PSK из истории git была выполнена
`git filter-repo --replace-text repl.txt --force --path site/etc/hostname.iwm0`.
Флаг `--path` в filter-repo — это фильтр **«оставить только этот путь»**, а не
«ограничить замену этим файлом». Команда выкинула из истории и рабочего дерева
ВСЁ, кроме `site/etc/hostname.iwm0`. `gc` уже прошёл → исходные git-объекты
утрачены безвозвратно (fsck --lost-found пуст; на GitHub был запушен битый
2-коммитный остаток).

## Что уцелело и стало источниками восстановления
- **`dist/velo79.img`** (Jun-13) — install-образ. Из его FFS-партиции (UFS2)
  извлечён `7.9/amd64/bsd.rd`; распакован gzip-ramdisk (FFS1); из ramdisk —
  **`/velo-install` (99517 B) и `/velo-tui.ksh` (21113 B)** — это и есть
  авторитетные текущие версии (Jun-13). Чтение FFS — самописный UFS1/UFS2-ридер
  (Linux не монтирует OpenBSD FFS).
- **`dist/site79.tgz`** (Jun-13) — `make-site-tgz.sh` делает `cp -R site/.`, то
  есть тарбол = 1:1 копия `site/`. Весь `site/` восстановлен побайтно отсюда.
- **JSONL-транскрипты сессий** — хронологический реплей (Read как снапшот, Edit
  как патч) восстановил `build/`, `docs/`, `demo/`, тесты-обёртки.
- **`/home/thx1138/velo-mut.3SUdmC/velo`** (Jun-5, mut-копия) — база для
  `tests/velo-install-test.ksh` и `tests/site-validate.ksh` (их в транскриптах
  никогда не читали целиком) + `build/dot.profile.hook`, `_pkgs/.keep-empty`.
- **gitignored выжившие на диске:** `vm/` (356), `dist/`, `site/usr/obj/_pkgs/*.tgz`.
