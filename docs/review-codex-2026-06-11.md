# Comments from Codex

Дата оценки: 2026-06-11.

## Пересмотренная оценка

`velo` нельзя оценивать только по качеству тестов. Как установочная технология
он реализован хорошо, но как готовая система для повседневного использования
пока заметно слабее.

## Что сделано хорошо

- Собственный TUI, работающий в ограниченном OpenBSD ramdisk.
- Защищённая установка на softraid CRYPTO.
- BIOS, UEFI и hybrid boot.
- Многоуровневая защита от выбора неправильного диска.
- Офлайн-замыкание пакетов.
- Проверка входных данных и защита от shell injection.
- Очистка passphrase и паролей из состояния.
- Хорошая история практических VM-проверок.

На момент оценки проходят:

- 104 встроенных assertions;
- 226 тестов `velo-install-test.ksh`;
- 115 проверок site-слоя.

Это подтверждает качество установщика, но не полноту продукта после установки.

## Главный недостаток: что происходит после установки

После завершения установки пользователь получает OpenBSD/XFCE с выбранным
начальным профилем. Полноценный жизненный цикл системы ещё не реализован:

- переключатель L1/L2/L3 отложен на v0.2;
- нет законченного Control Center;
- нет удобного обновления с проверкой и откатом;
- нет понятного recovery workflow для обычного пользователя;
- L3 является SOCKS-only и требует правильной настройки приложений;
- нет изоляции приложений или отдельного gateway;
- desktop-профиль не образует самостоятельную законченную ОС уровня SecBSD.

Поэтому проект сильнее как безопасный OpenBSD installer framework, чем как
готовый privacy-дистрибутив.

## Сравнение с `install18.img`

Локальный `install18.img` уже использует гибридную схему: ESP начинается с
сектора 64, OpenBSD-раздел `0xa6` с сектора 1024. `velo` силён в разметке,
шифровании, выборе цели и воспроизводимости установки.

Но готовая SecBSD/XFCE-система потенциально выигрывает в области, которая
начинается после первого boot: интегрированный desktop, инструменты управления,
обновления и ежедневный пользовательский сценарий.

## Рекомендуемое направление

Не пытаться сразу строить ещё один полноценный privacy-дистрибутив. Лучше
зафиксировать узкую сильную роль:

> Velo — безопасный интерактивный установщик зашифрованного OpenBSD с
> воспроизводимыми профилями и guarded disk operations.

Дальнейшие варианты:

1. Использовать `velo` как installer framework для более зрелого desktop-слоя.
2. Сделать профиль установки SATYR gateway.
3. Импортировать или адаптировать проверенный SecBSD desktop вместо создания
   всего пользовательского слоя с нуля.
4. Отдельно разработать post-install lifecycle: control center, update,
   rollback, diagnostics и recovery.

## Приоритеты

1. Переписать README как честное описание текущего продукта.
2. Определить минимальный законченный post-install сценарий.
3. Реализовать безопасный переключатель профилей или удалить обещание v0.2 из
   пользовательского позиционирования до его готовности.
4. Добавить update/recovery runbook и автоматическую диагностику.
5. Провести физический acceptance test на выделенном SSD.
6. Исправить владельцев файлов UID/GID 65534.

## Оценка

- Установочная технология: **8.5/10**.
- Готовая пользовательская система: **5-6/10**.
- Проект целиком в текущем состоянии: **около 7/10**.
