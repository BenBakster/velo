## 1. Карта дисков этой машины (критично!)

**Букве `sdX` НЕ доверять** — порядок присвоения зависит от того, что воткнуто. Опознавать
цель ТОЛЬКО по содержимому (размер + removable + transport=usb), как делает guard-стек.

| Узел | Что это | Можно писать? |
|---|---|---|
| **nvme0n1** | внутренний SSD — **ЕДИНСТВ. системный диск Void** (грузимся отсюда) | **НИКОГДА** |
| **sda** (исторически) | Samsung 870 EVO **250 ГБ** в кармане TOSHIBA — теперь **Void-portable клон** (загрузочный LUKS, `project-void-portable-clone-sda`) | **НЕТ** (не USB; 250 ГБ; загрузочная система) |
| **sdX (новый)** | **чистая USB-флешка** под velo-образ — появляется при втыкании | **ДА — только это** |

Различитель цели = **removable (RM=1)** + **TRAN=usb** + **размер в окне 20–64 ГБ**, и цель
НЕ несёт корень (`findmnt /`). nvme — система; 250 ГБ Samsung — Void-portable клон, не цель.

## 2. Идентификация цели (Void/Linux) — повторяем guard-стек write-usb.sh вручную

1. **До** втыкания USB запиши текущие диски:
   ```sh
   lsblk -dno NAME,SIZE,MODEL,TRAN
   ```
2. Воткни 30 ГБ USB, повтори ту же команду. **Новый** узел — твоя цель. Сверь:
   - размер ≈ **28–30 GiB** (не 250 ГБ, не размер nvme);
   - `TRAN` = `usb`; `lsblk -o NAME,RM` показывает **RM=1** (removable).
   ```sh
   lsblk -o NAME,SIZE,TYPE,RM,TRAN,MOUNTPOINTS,MODEL
   ```
3. **Guard «не система»:** убедись, что корень НЕ на цели:
   ```sh
   findmnt -no SOURCE /        # должно быть что-то вроде /dev/nvme0n1pN — НЕ твой sdX
   ```
4. **Guard «размонтирован»:** у цели не должно быть смонтированных разделов. Если есть —
   отмонтируй (НЕ форматируй):
   ```sh
   lsblk /dev/sdX                      # посмотреть разделы и mountpoints
   sudo umount /dev/sdX*  2>/dev/null   # отмонтировать все раз­делы цели
   ```
5. **Guard «размер/вместимость»:** образ 2.0 GiB обязан влезть (30 ГБ >> 2 ГБ ✓), и цель
   НЕ должна быть «слишком большой» (>256 ГБ = почти наверняка системный диск → стоп).

> ⚠️ Подставляй реальное имя вместо `sdX`. Перепутать `sdX` с `sda`(250 ГБ) или `nvme0n1`
> — значит уничтожить данные/систему. Семь раз проверь по размеру.

## 3. Запись (Void/Linux) — основной путь = `flash-sda-guarded.sh`

`write-usb.sh` — canonical-скрипт, но он для **OpenBSD** (`/dev/rsd2c`, `disklabel`). На
Void основной путь — **`build/flash-sda-guarded.sh`**: тот же guard-стек, что и write-usb
(RM/TRAN/размер/не-root/не-nvme/размонтирован), плюс dry-run по умолчанию, typed-confirm и
post-write `cmp`-верификация. Устройство — **обязательный аргумент, дефолта нет**.

```sh
cd ~/Документи/_Проекты/velo
# ФИНАЛЬНАЯ сверка цели прямо перед записью:
lsblk -o NAME,SIZE,RM,TRAN,MOUNTPOINTS /dev/sdX

# СУХОЙ ПРОГОН (без VELO_I_AM_SURE) — прогоняет все гарды, ничего не пишет:
sudo build/flash-sda-guarded.sh dist/velo79.img /dev/sdX

# РЕАЛЬНАЯ ЗАПИСЬ (необратимо!) — попросит впечатать basename устройства:
sudo VELO_I_AM_SURE=yes build/flash-sda-guarded.sh dist/velo79.img /dev/sdX
```
- `/dev/sdX` — **весь диск**, не раздел (`/dev/sdX1` — НЕПРАВИЛЬНО).
- Скрипт сам делает `conv=fsync` + `sync` + drop_caches + `cmp` (байтовая сверка).

### Фолбэк — ручной dd (если скрипт недоступен)
```sh
sudo dd if=dist/velo79.img of=/dev/sdX bs=4M oflag=sync status=progress; sync
```

Проверка после записи:
```sh
sudo partprobe /dev/sdX 2>/dev/null; lsblk /dev/sdX   # должны появиться разделы velo (MBR 0xA6 / EFI)
```

### Альтернатива — canonical OpenBSD-путь (если пишешь из OpenBSD с USB-passthrough)
```sh
# на OpenBSD-боксе/ВМ, где USB виден как sdN (sysctl hw.disknames):
VELO_I_AM_SURE=yes build/write-usb.sh dist/velo79.img sd2
```
Скрипт сам прогонит 7 guard'ов (root-диск, размонтирован, размер ≤256 ГБ, существование)
и попросит **повторно впечатать имя устройства**. Без `VELO_I_AM_SURE=yes` — сухой прогон.

## 4. После записи — загрузка и установка

1. Воткни USB в целевой ноут, в BIOS/boot-menu выбери загрузку с USB (legacy/SeaBIOS или
   UEFI — velo поддерживает оба; мастер сам определит прошивку).
2. Появится **ASCII-TUI velo** (синий бокс). Мастер (9 экранов при FDE):
   диск → шифрование → пароль(и) → hostname → **профиль = desktop** → пакеты →
   **Protection level** (L1 baseline / L2 / L3 Tor-only) → boot-mode → summary →
   DESTROY-гейт (впечатать имя диска).
3. **Целевой диск установки** выбирай ОТДЕЛЬНЫЙ (рекоменд. — внешний SSD), **не сам USB**
   и **не системный nvme**. Мастер до деструктива покажет идентичность диска (размер/DUID).
4. Установка идёт оффлайн (пакеты из встроенного closure). На **первой загрузке** цели
   `pkg_add` поставит десктоп (≈235 пакетов) из closure — это пара минут, экран может
   показывать прогресс/паузу. Дальше — xenodm → XFCE.

**Что должно быть на установленном десктопе (проверка задачи B):**
- XFCE + Firefox (chromium нет — firefox-only);
- скриншот: `xfce4-screenshooter` (PrintScreen / через меню);
- `xfce4-power-manager` (батарея), `xfce4-clipman` (буфер), `xfce4-whiskermenu` (меню);
- `xarchiver` (распаковка в Thunar), `zathura` (PDF);
- WiFi сам цепляется к преднастроенным WiFi-сетям (join-список).

> Примечание: пароли в мастере ты задаёшь сам. Тестовые креды из vm/-драйверов
> (`velofde9X` и т.п.) в образ НЕ зашиты — это входы автотеста, не дефолты.

## 5. Если что-то не так

- **Не грузится с USB:** проверь порядок загрузки в BIOS; для очень старого железа в
  мастере есть boot-mode `bios` (legacy MBR) — но это про целевой диск, не про USB.
- **cksum образа не совпал:** пересобрать (fetch-pkg-closure → make-site-tgz → grow-media
  в buildhost.qcow2; рецепт — docs/WORKLOG.md «VM-сборка», сессия 12; транспорт —
  `vm/ssh_pw.py`, root@127.0.0.1:2222, пароль `velobuild79`).
- **Сомнение в цели — СТОП.** Лучше переспросить, чем перезаписать sda(250 ГБ) или nvme.
